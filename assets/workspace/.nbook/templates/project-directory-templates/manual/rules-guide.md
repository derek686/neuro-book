# 规则指南

本文件是当前 Project 的人类可读规则参考。它说明游戏如何裁决行动；详细 canon、完整公式和系统表仍放在 `lorebook/`。

## Tick 流程

一次行动可以按 tick 推进：

1. 主持人描述当前处境。
2. 玩家说明角色想做什么。
3. 世界按规则回应这次尝试。
4. 主持人裁定结果，并描述新的局面。

玩家输入代表尝试，不代表结果已经发生。

## 行动裁决

说明哪些行动可以自动成功，哪些行动需要判定，哪些行动因为世界规则或已确认事实而不可能。

常用检查项：

- 能力与训练。
- 资源与装备。
- 时间与距离。
- 环境与阻碍。
- 风险与失败代价。

## 信息披露

说明开局、角色创建、场景推进和隐藏真相的披露边界。原则上，玩家只应获得角色当前能知道、感知或合理推断的信息；隐藏真相通过线索、代价和行动确认逐步揭示。

## 难度

写出本 Project 的简明难度梯度，例如自动成功、简单、标准、困难、极难和不可能。

## 资源

列出玩家和主持人需要追踪的资源，例如生命、体力、魔力、金钱、时间、声望、补给或特殊点数。

## 探索、社交与战斗

简述本桌如何处理：

- 探索与线索。
- 社交行动与 NPC 反应。
- 战斗或危险场景。
- 旅行与休整。
- 制作、训练或成长。

## 系统模块

列出游戏中可能启用的模块，例如经济、随机表、战斗面板、关系系统或特殊能力。本文件只说明玩法入口；需要来源时链接到 `lorebook/` 或 `manual/reference.md`。
